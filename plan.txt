We want to extend the Trip Media system with media visibility and friend sharing.

Goal:
Allow users to upload personal photos inside trips that are not visible publicly but can optionally be shared with selected friends.

Implementation plan should include:

1. Media visibility levels
Each TripMedia record must support:
PUBLIC
FRIENDS
PRIVATE

Default should remain PUBLIC.

2. Database updates
Add MediaVisibility enum and visibility field to TripMedia.

3. Friend system
Introduce a UserFriend relation table supporting friend requests and accepted friendships.

4. Media sharing
When visibility = FRIENDS the system should allow access only to:
- the owner
- accepted friends
- explicitly shared users.

Implement a MediaAccess table linking mediaId and userId.

5. Access control
All gallery APIs must enforce media visibility rules server-side.

6. UI updates
Add a visibility selector for each media item inside the trip media manager.

7. Friend management
Create a /friends page allowing users to send and accept friend requests.

Before implementation begins present the proposed schema and API endpoints for review.
